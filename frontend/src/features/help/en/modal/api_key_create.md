# Create API Key

Personal API keys (prefix `upk_`) allow external tools and scripts to access the portal API.

## Name
A memorable label to identify the key later.

## Scopes
Select which API scopes the key should have. Available scopes are grouped by resource:
- `cluster:read` – read cluster data
- `jobs:read` / `jobs:write` – list and start jobs
- `playbooks:read` – list playbooks
- `owners:read` – read ownership data

## Expiry
Optionally set an expiry date. The key will stop working after this date.

## After creation
The full key is shown **once**. Copy it now – you cannot retrieve it later.

> Core edition: max. 1 API key per user. Plus: unlimited.
