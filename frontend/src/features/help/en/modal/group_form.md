# Create / Edit Group

Groups bundle users together for easier permission management.

## Group name
Choose a unique, descriptive name (e.g. `team-ops`, `devs-berlin`).

## Members
Add existing local portal users to the group. Changes take effect immediately for the user's next request.

## Tags
Optional labels for filtering and organisation.

## Inherited permissions
The group can be assigned a role preset and node access rights. All group members inherit these permissions.

> **Note:** Core edition: max. 3 groups.
