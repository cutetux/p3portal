# Automation – Playbooks

Führe Ansible-Playbooks für Infrastruktur-Konfiguration und Management-Aufgaben aus (nicht vom Deployment-Bereich abgedeckt).

## Unterschied zu Deployment
Während der Deployment-Bereich auf das Erstellen neuer VMs/LXCs ausgerichtet ist, werden Automation-Playbooks für Rekonfiguration, Updates und operative Aufgaben an bestehenden Ressourcen genutzt.

## Playbook ausführen
1. Playbook aus der Liste links auswählen.
2. Parameter im Formular rechts ausfüllen.
3. Auf **Ausführen** klicken – das Portal startet einen Ansible-Job und streamt die Live-Ausgabe.

## Playbooks hochladen
Administratoren können neue Playbooks als ZIP-Archiv hochladen. Das Archiv muss das Playbook-Verzeichnis mit `meta.yaml` auf oberster Ebene enthalten.

## Berechtigungen
Sichtbare Playbooks hängen von deiner Rolle und konfigurierten Playbook-Whitelists ab.
